## Algorithm — ALS and the ALS-XZ Rule

### Almost Locked Set (ALS)

An **Almost Locked Set** of size N in a unit is a set of N cells in that unit such that the union of their candidates contains exactly N+1 distinct digits. Equivalently: "one more digit than cells." If any external constraint eliminates ANY single digit from these N cells, the remaining N digits lock into the N cells (becoming a Locked Set / Hidden N-Tuple).

Example (size 2, candidates {1,2,3}): cells (0,0)={1,2}, (0,1)={2,3} in row 0. Union {1,2,3}. If something forces digit 1 OUT of these cells, then {2,3} must occupy these two cells, so digit 2 lives in one of them (and digit 3 in the other).

We enumerate ALSes of size 2 through 5 (size 1 is just a bivalue cell, already handled by the chain graph; size > 5 is computationally expensive with diminishing returns). Per the spec.

### The ALS-XZ Rule

Given two **disjoint** ALSes `A` and `B` (no shared cells) such that:
1. They share at least two candidates (digits `X` and `Z`).
2. Exactly one of those shared digits, `Z`, is a **restricted common candidate (RCC)** — every Z-bearing cell in A "sees" (shares a unit with) every Z-bearing cell in B.

…then the other shared digit `X` can be eliminated from any cell that is **not in A or B** AND **sees every X-bearing cell in both A and B** AND has X as a candidate.

**Why it works:** Either `Z` lives in A (locking A, releasing X to occupy a specific cell in A), or `Z` lives in B (locking B, releasing X in B). In EITHER case, X must appear in one of A's or B's X-bearing cells. So any external candidate that sees ALL of those X-cells cannot itself be X — it would force X out of both A and B simultaneously, but we just proved X must live in one of them.

### Implementation choices

- **Enumerate ALSes from every unit** (rows, cols, boxes, diagonals, cages) — variant-aware via the existing `build_units`. Sizes 2 through 5. Skip cells with 0 candidates (filled).
- **For ALS-XZ pairs**, require disjoint cells (no shared cells between A and B). Overlapping ALSes give the same logical inference as their disjoint counterparts plus complications; we keep it simple.
- **Exactly one RCC.** ALS-XY-Wing (two RCCs) is a strict superset and adds another layer of enumeration. Skip for this commit; if Inkala still stucks, revisit.
- **First elimination wins.** Like `find_aic` and `find_bivalue_forcing`, return on the first ALS-XZ pair that yields any elimination. The outer solver loop will call us again next iteration on the reduced candidate state.

### Performance

- **Enumeration cost:** per unit (typically 9 cells), C(9,2)+C(9,3)+C(9,4)+C(9,5) = 36+84+126+126 = 372 subsets. Each subset checked in O(N). For 27 units (classic) + 0-30 cage units (killer): ~10,000 subset operations per board. Trivial.
- **ALS-XZ cost:** if `M` is the ALS count (typically 50-300 on Inkala-density boards), pairwise scan is `M^2 / 2`. Per pair: O(common_digits × cells^2) for RCC + O(CELLS) for elimination scan. Roughly 100 ops/pair → 1M-9M ops/board. Well under 500ms budget.
- **Lazy.** ALS only runs when T1-T4 + simple coloring + forcing chains have all failed. So most puzzles never trigger it.

// --- Tier 5: Almost Locked Sets (ALS-XZ) --------------------------------
//
// An ALS is N cells in a unit whose union of candidates contains exactly
// N+1 distinct digits. The "+1" makes it "almost" locked — removing any
// single digit collapses it into a hidden N-tuple.
//
// We enumerate ALSes of size 2..=5 across every unit (row, col, box, diag,
// cage) using the existing variant-aware `build_units`. The ALS-XZ rule
// (next function) consumes the enumerated list.

const ALS_MIN_SIZE: usize = 2;
const ALS_MAX_SIZE: usize = 5;

#[derive(Debug, Clone)]
struct Als {
    /// Cell indices (0..CELLS), sorted ascending. All cells live in the
    /// same unit (the one that produced this ALS).
    cells: Vec<usize>,
    /// Bitmask of candidate digits across all cells (always popcount == cells.len() + 1).
    candidates: u16,
    /// digit_cells[d-1] = cells in this ALS that carry digit d. Empty if d
    /// is not in this ALS's candidate union.
    digit_cells: [Vec<usize>; 9],
}

/// Iterate every size-`k` subset of `pool` (k <= pool.len()) and call `f`
/// with each subset (as borrowed indices into `pool`). Standard "next
/// combination" iteration — no allocation per subset beyond the index Vec.
fn for_each_combination<F: FnMut(&[usize])>(pool: &[usize], k: usize, mut f: F) {
    let n = pool.len();
    if k == 0 || k > n {
        return;
    }
    let mut idx: Vec<usize> = (0..k).collect();
    let mut buf: Vec<usize> = vec![0; k];
    loop {
        for (j, &i) in idx.iter().enumerate() {
            buf[j] = pool[i];
        }
        f(&buf);
        // Advance to next combination, lex order.
        let mut i = k;
        loop {
            if i == 0 {
                return;
            }
            i -= 1;
            if idx[i] < n - (k - i) {
                idx[i] += 1;
                for j in (i + 1)..k {
                    idx[j] = idx[j - 1] + 1;
                }
                break;
            }
        }
    }
}

fn find_alses(c: &Candidates, units: &[Unit]) -> Vec<Als> {
    let mut alses: Vec<Als> = Vec::new();
    let mut active_cells: Vec<usize> = Vec::with_capacity(9);
    for u in units {
        active_cells.clear();
        for &(r, col) in &u.cells {
            let i = r * N + col;
            if c.masks[i] != 0 {
                active_cells.push(i);
            }
        }
        if active_cells.len() < ALS_MIN_SIZE {
            continue;
        }
        let max_size = ALS_MAX_SIZE.min(active_cells.len());
        for size in ALS_MIN_SIZE..=max_size {
            for_each_combination(&active_cells, size, |subset| {
                let mut union: u16 = 0;
                for &i in subset {
                    union |= c.masks[i];
                }
                if popcount(union) as usize != size + 1 {
                    return;
                }
                let mut digit_cells: [Vec<usize>; 9] = Default::default();
                for &i in subset {
                    let m = c.masks[i];
                    for d in 1u8..=9 {
                        if m & bit(d) != 0 {
                            digit_cells[(d - 1) as usize].push(i);
                        }
                    }
                }
                alses.push(Als {
                    cells: subset.to_vec(),
                    candidates: union,
                    digit_cells,
                });
            });
        }
    }
    alses
}
```

## Task 4: Implement `find_als_xz` with smoke test

**Files:**

**Algorithm recap:**

For each pair `(A, B)` of disjoint ALSes:
1. Find common digits (intersection of candidate masks).
2. If fewer than 2 common digits, skip (need 1 RCC + 1 X).
3. For each common digit Z, check if it's restricted: every Z-bearing cell in A must share a unit with every Z-bearing cell in B. Use the `PeerTable` (cell `a` sees cell `b` iff `peers.peers[a].contains(&b)`).
4. If exactly 1 RCC Z, proceed; otherwise skip.
5. For each non-Z common digit X: find cells outside A∪B that have X as a candidate AND see EVERY X-bearing cell in A AND EVERY X-bearing cell in B. Those are X-eliminations.
6. Return the first ALS-XZ pair with at least one elimination.

fn find_als_xz(c: &Candidates, peers: &PeerTable, units: &[Unit]) -> Option<Step> {
    let alses = find_alses(c, units);
    if alses.len() < 2 {
        return None;
    }
    for i in 0..alses.len() {
        for j in (i + 1)..alses.len() {
            let a = &alses[i];
            let b = &alses[j];
            // Require disjoint cells.
            if a.cells.iter().any(|ai| b.cells.contains(ai)) {
                continue;
            }
            let common = a.candidates & b.candidates;
            if popcount(common) < 2 {
                continue;
            }
            // Find RCCs.
            let mut rcc: Option<u8> = None;
            let mut multi_rcc = false;
            for d in 1u8..=9 {
                if common & bit(d) == 0 {
                    continue;
                }
                let za = &a.digit_cells[(d - 1) as usize];
                let zb = &b.digit_cells[(d - 1) as usize];
                if za.is_empty() || zb.is_empty() {
                    continue;
                }
                let mut restricted = true;
                'pairs: for &ca in za {
                    for &cb in zb {
                        if !peers.peers[ca].contains(&cb) {
                            restricted = false;
                            break 'pairs;
                        }
                    }
                }
                if restricted {
                    if rcc.is_some() {
                        multi_rcc = true;
                        break;
                    }
                    rcc = Some(d);
                }
            }
            if multi_rcc || rcc.is_none() {
                continue;
            }
            let z = rcc.unwrap();
            // For each non-Z common digit X, scan for eliminations.
            let mut removed: Vec<(usize, usize, u8)> = Vec::new();
            for x in 1u8..=9 {
                if x == z || common & bit(x) == 0 {
                    continue;
                }
                let xa = &a.digit_cells[(x - 1) as usize];
                let xb = &b.digit_cells[(x - 1) as usize];
                if xa.is_empty() || xb.is_empty() {
                    continue;
                }
                for victim in 0..CELLS {
                    if c.masks[victim] & bit(x) == 0 {
                        continue;
                    }
                    if a.cells.contains(&victim) || b.cells.contains(&victim) {
                        continue;
                    }
                    let sees_all_in_a = xa.iter().all(|&cell| peers.peers[victim].contains(&cell));
                    if !sees_all_in_a {
                        continue;
                    }
                    let sees_all_in_b = xb.iter().all(|&cell| peers.peers[victim].contains(&cell));
                    if !sees_all_in_b {
                        continue;
                    }
                    let elim = (victim / N, victim % N, x);
                    if !removed.contains(&elim) {
                        removed.push(elim);
                    }
                }
            }
            if !removed.is_empty() {
                return Some(Step::Elimination {
                    technique: Technique::Als,
                    removed,
                });
            }
        }
    }
    None
}
```

- Common digits {1, 3}, of which exactly one (Z=1) is restricted.
- A victim cell outside A and B that sees every X=3 bearer in A and B.

**Setup:**
- `(0,0) = {1, 2}` — ALS A cell 1 (row 0).
- `(0,1) = {2, 3}` — ALS A cell 2 (row 0).
- `(3,0) = {1, 3}` — ALS B cell 1 (col 0).
- `(5,0) = {3, 4}` — ALS B cell 2 (col 0).
- `(1,0) = {3, 6}` — VICTIM (digit 3 to be eliminated).
- All other cells: empty mask (no candidates).

**Verification trace:**
- ALS A = row 0 cells `[(0,0), (0,1)]`, union `{1,2,3}`. Size 2, 3 candidates. ✓ ALS.
- ALS B = col 0 cells `[(3,0), (5,0)]`, union `{1,3,4}`. Size 2, 3 candidates. ✓ ALS.
- Disjoint cells. ✓
- Common digits: `{1, 3}`. 2 of them. ✓ ≥ 2.
- RCC check:
  - Z=1: A's 1-cells = `[(0,0)]`. B's 1-cells = `[(3,0)]`. (0,0) sees (3,0) via col 0. ✓ Restricted.
  - Z=3: A's 3-cells = `[(0,1)]`. B's 3-cells = `[(3,0), (5,0)]`. (0,1) sees (3,0)? Different row, col 1 vs col 0, box 0 vs box 3. NO. Not restricted.
  - Exactly 1 RCC: Z=1. ✓
- X = 3 (the other common digit).
- X-cells in A: `[(0,1)]`. X-cells in B: `[(3,0), (5,0)]`.
- Victim scan: candidate cells with digit 3 (other than ALS cells) → only `(1,0):3` in our fixture.
- Does (1,0) see (0,1)? Same box 0 (rows 0-2, cols 0-2). YES.
- Does (1,0) see (3,0)? Same col 0. YES.
- Does (1,0) see (5,0)? Same col 0. YES.
- → Eliminate `(1,0):3`.

The test asserts `(1,0,3) ∈ removed`.

**Caveat:** other ALSes also exist in this fixture (e.g. (0,0)+(3,0) in col 0 = ALS {1,2,3}, (1,0)+(3,0) in col 0 = ALS {1,3,6}, etc.). The algorithm iterates pairs and returns on first elimination. The pair we designed should be the first to fire IF it sorts earlier than other valid pairs. To make the test robust regardless of iteration order, we only assert `(1,0,3)` is AMONG `removed`, not that it's the ONLY entry. The algorithm only returns one Step at a time (from one ALS pair), so `removed` contains eliminations from one pair only.

If the first-firing pair produces eliminations that don't include `(1,0,3)`, the test will fail and we'll need to inspect which pair won. Most likely fix: tighten the fixture by clearing more digits.
