# Ship 16×16 (Classic + X-Sudoku) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Expose 16×16 puzzles for Classic and X-Sudoku end-to-end (server acceptance + web rendering/input/selector), riding out with the parked `solver-constraint-propagation` branch as one deploy.

**Architecture:** The engine is already 16×16-capable (`--size 16`, digits 10–16 emitted as `A`–`G`, sub-second generate+grade at the 47% clue floor). **Tech Stack:** Rust engine binaries (`stillgrid-generate`, `stillgrid-grade`); Node + Express + TypeScript server (vitest); React 18 + Vite + TypeScript web (vitest).

**Spec:** `docs/superpowers/specs/2026-06-02-16x16-ship-design.md`

## Task 1.5: Fix `fill_random` — robust 16×16 generation (MRV)

**Why:** `engine/src/generator.rs` `fill_random` builds the initial complete solution with naive first-empty backtracking + `can_place` checks and no propagation. At 256 cells this backtracks catastrophically on ~5% of seeds (measured: classic@16 seeds 16 and 28 of 1–40 exceed 3 s; seed 28 > 30 s), so the server's 15 s `generate` timeout fires and the request 500s. Switching cell selection to **MRV** (fewest-candidates-first) collapses the search to near-linear and eliminates the hang. This is the roadmap's flagged `fill_random` fix; it also unblocks the deferred Jigsaw@16.

**Files:**
- Modify: `engine/src/generator.rs` — rewrite `fill_random` (lines ~39-56); remove now-dead `first_empty` (lines ~58-67)
- Test: `engine/src/generator.rs` `#[cfg(test)]` module — add a bound-checked 16×16 generation test

**Context:** `Rng::shuffle<T>(&mut self, &mut [T])` shuffles in place. `Variant::can_place(&self, board, r, c, v: u8) -> bool`. `Board::get(r, c) -> u8`, `Board::set(r, c, v)`. Generated puzzles need not be byte-for-byte stable (no golden test pins generator output, and dailies are regenerated per request with no content-keyed persistence) — the only invariants are: produces a *valid, complete, unique-carvable* solution, and stays fast. 9×9/6×6 must keep working (they already never hang; MRV only changes RNG consumption, which no test pins).

```rust
fn fill_random(board: &mut Board, variant: &Variant, rng: &mut Rng) -> bool {
    let n = variant.n as usize;
    // MRV: choose the empty cell with the fewest legal candidates. This collapses
    // the 256-cell search from catastrophic backtracking (naive first-empty) to
    // near-linear, eliminating the 16×16 generation hang. Digit order stays random
    // for solution diversity.
    let mut best: Option<(usize, usize, u32)> = None;
    let mut best_count = usize::MAX;
    'scan: for r in 0..n {
        for c in 0..n {
            if board.get(r, c) != 0 {
                continue;
            }
            let mut cands: u32 = 0;
            let mut count = 0usize;
            for val in 1..=n as u8 {
                if variant.can_place(board, r, c, val) {
                    cands |= 1 << val;
                    count += 1;
                }
            }
            if count == 0 {
                return false; // dead cell — fail this branch fast
            }
            if count < best_count {
                best_count = count;
                best = Some((r, c, cands));
                if count == 1 {
                    break 'scan; // a forced cell — can't do better
                }
            }
        }
    }
    let Some((r, c, cands)) = best else {
        return true; // no empty cell remains → grid complete
    };
    let mut digits = [0u8; 16];
    let mut k = 0;
    for val in 1..=n as u8 {
        if cands & (1 << val) != 0 {
            digits[k] = val;
            k += 1;
        }
    }
    rng.shuffle(&mut digits[..k]);
    for &val in &digits[..k] {
        board.set(r, c, val);
        if fill_random(board, variant, rng) {
            return true;
        }
        board.set(r, c, 0);
    }
    false
}
```

```ts
/** Digit → display char: 1–9 decimal, 10–16 → 'A'–'G'. Matches engine output. */
export function digitToChar(d: number): string {
  return d <= 9 ? String(d) : String.fromCharCode(55 + d); // 10→'A' … 16→'G'
}

/** Display/wire char → digit (1–16); 0 for empty/invalid. Accepts a–g and A–G. */
export function charToDigit(ch: string): number {
  if (ch >= "1" && ch <= "9") return ch.charCodeAt(0) - 48;
  const up = ch.toUpperCase();
  if (up >= "A" && up <= "G") return up.charCodeAt(0) - 55; // 'A'→10 … 'G'→16
  return 0;
}
```
